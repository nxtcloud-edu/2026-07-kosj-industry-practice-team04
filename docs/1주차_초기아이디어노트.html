01

문제 정의

기존 신고 체계는 시민에게 복잡하고, 담당자에게는 중복 업무를 만든다.

CITIZEN

신고 접근성이 낮다

앱 설치, 로그인, 긴 양식 작성, 신고 유형 선택 등으로 디지털 취약층이 쉽게 이용하기 어렵다.

OFFICER

유사 신고가 흩어진다

같은 위치의 같은 문제가 여러 번 신고되어도 각각 별개의 민원으로 쌓여 중복 확인이 필요하다.

SYSTEM

중요한 문제가 묻힌다

접수 순서 중심 처리로 인해 위험도가 높거나 반복적으로 신고되는 문제가 뒤로 밀릴 수 있다.

시민의 입력은 최소화하고, 반복 신고는 하나의 문제로 모으며, 담당자는 실제로 먼저 처리해야 할 문제부터 확인하게 한다.
02

제안 서비스와 전체 흐름

React 기반 웹앱에서 신고부터 처리 상태 확인까지 하나의 흐름으로 제공한다.

STEP 01

웹앱 접속

QR코드나 링크로 접속하며 별도 앱 설치와 회원가입이 필요 없다.

STEP 02

사진 촬영

촬영 가이드와 음성 안내를 제공하고 사진은 자동 압축한다.

STEP 03

위치 확인

GPS로 현재 위치를 입력하고 사용자는 지도에서 수정할 수 있다.

STEP 04

AI 유형 분류

이미지 인식 API가 도로 파손, 가로등, 쓰레기 등으로 1차 분류한다.

STEP 05

유사 신고 통합

반경 50m 내 동일 유형 신고를 하나의 대표 문제로 묶고 누적 건수를 표시한다.

STEP 06

우선순위·배정

위험도와 신고 빈도로 우선순위를 정하고 유형·구역별 담당 부서를 자동 배정한다.

STEP 07

담당자 검수

오분류, 잘못된 통합, 담당 부서 및 우선순위를 관리자 화면에서 수정한다.

STEP 08

상태 확인

접수, 배정, 처리 중, 완료 단계를 확인하고 연락처 입력 시 결과 알림을 받는다.

03

접근성 중심 신고 설계

디지털 취약층, 고령자, 외국인, 저속 인터넷 이용자까지 고려한다.

🔎

글자 크기 확대·고대비 모드

저시력 사용자와 고령층을 위해 큰 글자, 큰 버튼, 색상 대비 강화 기능을 제공한다.

🔊

음성 안내

글을 읽거나 입력하기 어려운 사용자가 버튼을 누르면 사진 촬영부터 신고 완료까지 절차를 음성으로 안내한다.

①

3단계 진행 표시

사진 촬영 → 위치 확인 → 신고 완료를 화면 상단에 표시해 현재 단계를 명확하게 보여준다.

👤

로그인 없는 신고

회원가입과 로그인을 요구하지 않으며 연락처는 처리 결과 알림을 원하는 경우에만 선택적으로 입력한다.

✏️

신고 수정·취소

사진, 위치, 설명을 잘못 입력했을 때 신고 전 수정하거나 접수 후 일정 시간 안에 취소할 수 있도록 한다.

🌐

다국어 지원

한국어 외에 영어, 중국어, 베트남어 등 주요 언어로 핵심 안내와 신고 화면을 제공한다.

📶

저속 인터넷·연결 끊김 대응

사진을 자동 압축하고 작성 내용을 기기에 임시 저장해 통신이 복구되면 다시 전송할 수 있도록 한다.

🙋

‘나도 불편해요’ 기능

기존 유사 신고가 있으면 새 신고를 반복 작성하는 대신 버튼 한 번으로 기존 문제에 공감 건수를 추가한다.

📸

사진 촬영 가이드

문제 지점과 주변 환경이 함께 보이도록 예시 이미지, 촬영 문구, 재촬영 안내를 제공한다.

🔗

여러 신고 경로 통합

모바일 웹, 전화, 주민센터, 키오스크, 챗봇 등 여러 경로의 신고를 하나의 관리자 시스템으로 통합한다.

04

React 기반 웹앱과 시스템 구성

시민용 화면, 관리자 업무 화면, AI·데이터 처리 기능을 분리해 구성한다.

FRONTEND

React 기반 모바일 웹앱

  • 앱처럼 빠르게 화면 전환
  • 카메라 촬영과 GPS 위치 사용
  • 글자 확대, 고대비, 다국어 UI
  • 임시 저장과 처리 상태 조회
BACKEND

신고 처리 서버

  • 신고 데이터와 사진 저장
  • 유사 신고 거리 계산
  • 우선순위와 부서 자동 배정
  • 여러 신고 경로 데이터 통합
AI & DATA

이미지 분류·데이터 축적

  • 외부 이미지 인식 API 활용
  • 담당자 수정 결과를 라벨로 저장
  • 자체 학습 데이터셋 구축
  • 장기적으로 자체 모델 전환
05

유사 신고 통합과 우선순위 로직

중복 신고를 제거하지 않고 문제의 심각성을 판단하는 데이터로 활용한다.

유사 신고 통합 기준

  • 기존 신고와의 거리가 반경 50m 이내인지 확인
  • AI가 분류한 신고 유형이 동일한지 확인
  • 일정 기간 안에 접수된 신고인지 확인
  • 조건 충족 시 하나의 대표 문제 카드에 연결
  • 담당자가 잘못 통합된 신고를 분리할 수 있도록 제공

우선순위 산정 기준

  • 문제 유형별 기본 위험도 반영
  • 동일 위치의 누적 신고 건수 반영
  • ‘나도 불편해요’ 참여 수 반영
  • 향후 경과시간, 취약지역 여부를 추가 가중치로 활용
우선순위 = 기본 위험도 + 신고 빈도 + 공감 수
06

데이터 축적과 자체 모델 발전

초기에는 외부 API를 사용하고, 실제 운영 데이터를 바탕으로 자체 모델을 구축한다.

신고 이미지 수집
외부 API 1차 분류
담당자 검수·수정
정제된 라벨 데이터 축적
자체 분류 모델 고도화
모델 개선을 위해 추가 사진이 필요한 경우, 특정 신고자에게 여러 장의 사진 제공을 요청할 수 있다. 다만 이는 별도의 명확한 동의를 받은 사용자에게만 선택적으로 요청하며, 사진의 사용 목적과 보관 기간을 안내하고 언제든 참여를 거부할 수 있도록 한다.

08

MVP 구현 범위와 단계별 확장

모든 기능을 한 번에 구현하지 않고 핵심 흐름부터 완성한다.

PHASE 1 · MVP CORE

반드시 구현

React 모바일 웹, 사진·위치 신고, 3단계 진행 표시, 이미지 분류, 반경 50m 유사 신고 통합, 우선순위 산정, 담당 부서 배정, 관리자 검수, 처리 상태 관리

PHASE 2 · ACCESSIBILITY

우선 추가

글자 확대·고대비, 음성 안내, 다국어, 사진 촬영 가이드, 사진 자동 압축, 임시 저장, 신고 수정·취소, ‘나도 불편해요’, 연락처 선택 알림

PHASE 3 · ROADMAP

향후 고도화

전화·키오스크·챗봇 신고 통합, 지도·통계 대시보드, 얼굴·번호판 비식별, 추가 사진 참여 프로그램, 축적 데이터 기반 자체 이미지 분류 모델

시민에게는 설치와 로그인 없이 누구나 사용할 수 있는 신고 경험을, 담당자에게는 같은 문제를 모아 중요한 순서대로 보여주는 업무 환경을 제공한다.
