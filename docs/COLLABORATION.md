# 4팀 「모아」 협업 가이드 — GitHub 단일 체계

> 이슈·보드·코드·리뷰를 전부 GitHub 한 곳에서 관리합니다. 도구는 하나, 규칙은 단순하게.
> 명령어가 낯설면 → **[GIT_QUICKSTART.md](GIT_QUICKSTART.md)** 를 그대로 따라하세요.

**3대 원칙**

1. **이슈 없이는 작업 없음** — 모든 작업은 GitHub Issue로 시작한다
2. **main 직접 push 금지** — 모든 변경은 브랜치 → PR → CI → 머지
3. **main은 항상 시연 가능 상태** — CI(테스트 113개 + 빌드)가 필수 체크라 깨진 코드는 머지 자체가 안 된다

> 📌 **현재 운영 모드 (7/23~)**: 1인 개발 체제. 리뷰 승인 요건은 일단 해제 — **PR + CI 통과**가 머지 조건입니다. 팀 협업 재개 시 아래 리뷰 규칙(승인 1인·교차 리뷰)을 복원합니다.

## 1. 브랜치 전략 — 사람별 브랜치 (7/23 전환)

멘토 피드백: *"기능별 브랜치보다 사람별 브랜치가 속도 면에서 효율적"* — 3주차까지 쓰던 이슈별 브랜치(`12-photo-upload`)를 **사람별 브랜치**로 전환했습니다.

```
main ────●───────●─────●──→  항상 시연 가능 (CI 필수 체크 + 리뷰 1인)
          \     /     /
sunghyun   ●───●     /        ← 각자 자기 브랜치에서 작업, 몫이 쌓이면 PR
jaeyong     ●───────●
```

| 규칙 | 내용 |
|---|---|
| 브랜치 | `sunghyun` `jaeyong` `songeon` `dongjin` — 본인 것 하나만 유지 |
| 시작 전 | `git pull origin main` 으로 최신 main을 브랜치에 반영 (충돌 예방) |
| PR 단위 | 이슈 1~2개 몫. 커밋·PR에 이슈 번호(`Closes #21`)는 그대로 유지 |
| 머지 방식 | **Squash and merge** — main 히스토리가 "PR 1개 = 커밋 1개"로 유지 |
| 머지 후 | 브랜치를 지우지 않고 계속 사용 (`git pull origin main` 후 다음 작업) |

**main 보호 (적용 중)**: 직접 push 금지 · PR 필수 · **required checks: `backend-test` `frontend-build`** · force-push 금지 (승인 요건은 1인 개발 모드 동안 해제)

## 2. 작업 관리 — Issues + 칸반 보드

- 이슈는 화면 1개·API 1개 수준으로 잘게 (1~2일 크기), 제목은 명령형, 본문에 완료 조건
- 만들 때 **담당자 + 라벨 + 마일스톤** 지정
- **[칸반 보드](https://github.com/users/SungHyunC/projects/1)** (공개): `To do → In progress → In review → Done`
  - 조직 권한 제약으로 김성현 계정 소유 — 열람은 전원 가능, 편집 권한 필요하면 요청
  - 새 이슈·PR은 보드에 Add item으로 등록, 상태는 작업 시작/PR/머지 시점에 이동

**라벨**: 영역 `frontend` `backend` `ai` `admin` `infra` `보안` `발표` · 에픽 `mvp-1`~`mvp-5` `p0`
**마일스톤**: `3주차 스프린트`(완료) · `4주차 스프린트`(7/27~31)

## 3. 커밋 메시지 컨벤션

```
feat: 신고 위치 지도 핀 드래그 보정 (#7)
fix: GPS 권한 거부 시 세종시청 폴백 (#21)
docs: API 계약에 지오코딩 추가
refactor / test / chore
```

- 타입: `feat` `fix` `docs` `refactor` `test` `chore` · 기능·수정 커밋에는 이슈 번호

## 4. Pull Request 규칙

- **PR은 작게** — 이슈 1~2개 몫, 500줄 넘으면 쪼개기
- 본문 템플릿대로: What / Why / 확인 방법 / 스크린샷 + `Closes #N`
- **CI 초록 확인 후 리뷰 요청** — 빨간 PR은 리뷰 대상이 아님. 로컬 선확인: `cd backend && npm test`
- **리뷰는 24시간 내**, 교차 리뷰(FE↔BE), 코멘트 한 줄 이상 남기기
- 승인 1인이면 Squash merge (관리자는 긴급 시 bypass 가능 — 사후 공유 필수)
