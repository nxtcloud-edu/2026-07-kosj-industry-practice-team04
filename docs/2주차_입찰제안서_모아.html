제안 요약 — 한 장면으로 보는 「모아」

본 제안의 전부는 아래 한 장면으로 요약됩니다.

75세 시민도, 외국인 주민도, 바쁜 직장인도 — QR 접속 후 3단계 안에 신고를 마치고, 같은 불편은 중복 없이 하나로 모여 담당자에게 우선순위대로 전달되며, 신고자는 로그인 없이 처리 상태를 끝까지 확인한다.

3대 핵심 차별점

1

‘나도 불편해요’ 유사 신고 통합

반경·유형·시간 조건으로 유사 신고를 하나의 대표 문제로 통합. 중복 민원은 업무가 아니라 우선순위 근거 데이터가 됩니다. 신고자가 통합 여부를 직접 선택하고, 담당자는 오통합을 즉시 분리할 수 있습니다.

2

접근성이 MVP에 들어간 신고 경험

큰 글자 · 고대비 · 사진 촬영 가이드를 MVP 핵심 범위에 포함해, 디지털 취약층이 실제로 쓸 수 있는 신고 경험을 시연 단계에서 증명합니다. 로그인 없음 · 3단계 진행 표시가 기본값입니다.

3

데이터가 쌓일수록 좋아지는 플랫폼

담당자의 검수·수정 결과를 학습 라벨로 축적하고, 다양한 현장 사진 데이터를 수집해 분류 모델을 파인튜닝합니다. 운영할수록 정확해지고, 검수 비용은 줄어듭니다.

수행 원칙 — 완성도 ‘10’ 중 동작하는 ‘5’를 시연

RFP의 전체 요구사항을 완성형 ‘10’으로 정의하고, 사업 기간 내에는 핵심 절반인 ‘5’를 동작하는 MVP로 구현·시연합니다. 나머지 6~10 구간은 단계별 확장 로드맵(11p)으로 제시하며, 모든 기능은 RFP 요구사항 ID와 연결(12p)해 기술했습니다.

기존 신고 체계는 시민에게 복잡하고, 담당자에게는 중복 업무를 만든다.

RFP가 말하는 현재의 문제 (공고 1.2 문제점 분석)

단계RFP 진단현장에서 벌어지는 일

세 주체의 관점에서 다시 정의한 문제

CITIZEN — 시민

신고 문턱이 높다

앱 설치, 회원가입, 긴 양식, 신고 유형 직접 선택. 고령층·외국인 등 디지털 취약층은 신고를 포기하거나 전화·방문에 의존한다.

OFFICER — 담당자

유사 신고가 흩어진다

같은 위치의 같은 문제가 별개 민원으로 쌓여, 담당자는 처리보다 중복 확인에 시간을 쓴다. 중복 건수가 많다는 사실 자체는 활용되지 못한다.

SYSTEM — 시스템

중요한 문제가 묻힌다

접수 순서 중심 처리로 위험도·반복성이 반영되지 않는다. 데이터는 쌓이지만 분류 개선이나 예방 정비로 이어지지 않는다.

문제의 재정의

신고의 문턱을 낮추는 것만으로는 부족하다. 낮아진 문턱으로 늘어난 신고가 중복 업무가 아니라 우선순위 데이터가 되도록, 접수–통합–배정–확인의 전 과정을 하나의 흐름으로 다시 설계해야 한다.
제안 방향 3원칙 — ① 시민의 입력은 최소화한다(사진+위치면 충분) ② 반복 신고는 하나의 문제로 모아 심각도 신호로 쓴다 ③ 담당자는 먼저 처리해야 할 문제부터 본다

전체 사용자 흐름 — 시민은 3단계만 보고, 나머지는 시스템이 한다

STEP 01 · 시민

웹앱 접속

QR·링크로 접속. 앱 설치, 회원가입 없음

STEP 02 · 시민

사진 촬영

촬영 가이드 제공, 자동 압축 업로드

STEP 03 · 시민

위치 확인 → 완료

GPS 자동 입력 + 지도 수정. 접수번호 발급

STEP 04 · AI

유형 자동 분류

도로 파손·가로등·쓰레기 등 1차 판별

STEP 05 · AI

유사 신고 통합

반경·유형·시간 조건으로 대표 문제에 연결, 신고자가 최종 선택

STEP 06 · AI

우선순위·배정

위험도+빈도+공감 수로 산정, 유형·구역별 부서 자동 배정

검수·보정

오분류 수정, 오통합 분리, 우선순위 조정

핵심 기능 ① 유사 신고 통합과 우선순위

중복 신고를 지우지 않고, 문제의 심각성을 증명하는 데이터로 쓴다.

통합 판정 기준과 그 근거

조건초기값설정 근거
거리반경 50m 이내스마트폰 GPS의 도심 오차(통상 10~30m, 건물 음영 시 확대)를 감안해 오차 상한의 약 1.5~2배로 설정. 가로등·횡단보도 등 시설물 설치 간격(약 30~50m)과도 정합
유형AI 분류 동일 + 신뢰도 70% 이상신뢰도가 낮은 분류 결과로 자동 통합하면 오통합 위험이 커지므로, 임계값 미달 건은 통합 후보에서 제외하고 관리자 우선 검수 큐로 전달
시간최근 72시간 이내 접수 건동일 문제에 대한 재신고가 집중되는 초기 구간을 가정한 값으로, 통상적인 현장 확인·처리 리드타임(2~3일)을 함께 고려
※ 세 값은 모두 운영 파라미터로 설계합니다. 하드코딩하지 않고 관리자 설정으로 두어, 운영 데이터(분리율·재통합률)를 근거로 지속 보정합니다.
설계 원칙 — “자동 통합은 보수적으로, 분리는 쉽게” · 조건을 충족해도 시스템이 강제 통합하지 않습니다. 신고자에게 유사 신고 후보를 보여주고 ‘기존 문제에 추가’ 또는 ‘새 신고로 접수’를 직접 선택하게 하며(4p 화면 ③), 담당자는 언제든 통합을 해제할 수 있습니다.

오통합 분리 절차 — 잘못 묶였을 때

1비교 확인 — 담당자가 대표 문제 카드에서 통합된 개별 신고의 사진·위치·시각을 나란히 비교
2즉시 분리 — ‘분리’ 버튼 한 번으로 해당 신고를 독립된 새 문제 카드로 전환 (우선순위·배정 자동 재산정)
3사유 기록 — 분리 사유를 태그로 기록 (위치 상이 / 유형 상이 / 별개 문제) — 클릭 1회 수준의 최소 입력
4규칙 보정 — 분리 이력을 통합 파라미터(반경·시간·임계값) 보정의 근거 데이터로 축적 → 통합 정확도가 운영할수록 상승

우선순위 산정 — 담당자가 “먼저 할 일”을 보게 한다

우선순위 = 유형별 기본 위험도 + 누적 신고 빈도 + ‘나도 불편해요’ 공감 수
※ 확장 단계에서 경과 시간, 취약지역(어린이보호구역 등) 가중치를 추가합니다.
WHY IT WORKS

중복이 많을수록 순위가 오른다

기존 체계에서 중복 신고는 ‘지워야 할 업무’였지만, 「모아」에서는 통합된 신고 건수와 공감 수가 그대로 심각성의 증거가 되어 우선순위를 끌어올립니다. 시민의 반복 신고가 처음으로 의미를 갖게 됩니다.

“문제와 주변이 함께 보이게”라는 예시 이미지·문구를 촬영 전 제시하고, 흐리거나 너무 가까운 사진은 재촬영을 안내합니다. 분류 정확도와 직결되는 기능입니다.

A11Y · MVP

① 3단계 진행 표시 · 로그인 없음

사진 → 위치 → 완료의 현재 단계를 항상 상단에 표시하고, 회원가입·로그인을 요구하지 않습니다. 연락처는 알림을 원할 때만 선택 입력합니다.

음성 안내·다국어·임시 저장(저속 인터넷 대응)은 확장 로드맵 6~7단계에서 이어집니다. 접근성의 뼈대(큰 글자·고대비·가이드·3단계·무로그인)를 MVP에서 완성하고, 감각·언어 확장을 덧붙이는 순서입니다.

HOW

조회 방식

신고 완료 시 접수번호(SJ-2026-0716-1234)와 서명된 난수 토큰이 담긴 조회 전용 URL을 발급합니다. 조회는 접수번호+토큰 조합으로만 가능해 타인 신고의 무작위 열람이 차단됩니다. 토큰에는 개인정보가 포함되지 않으며, 처리 완료 후 6개월(사진·위치 보관 기간)까지 유효하고 이후 자동 무효화됩니다.

FALLBACK

URL을 잃어버리면

연락처를 남긴 신고자는 문자로 조회 링크를 다시 받을 수 있고, 남기지 않은 경우 완료 화면 저장·북마크를 안내합니다. 상태는 접수 → 배정 → 처리 중 → 완료 4단계로 표시됩니다(SFR-006).

전국 지자체 + 산하 공단

기초·광역 지자체 243곳과 시설관리공단·도시관리공단 등 산하기관. 생활 인프라 민원은 모든 지자체의 공통 업무이며, 안전신문고 기준 연간 수백만 건의 생활 신고가 발생합니다. ※ 추정

스마트시티 시범·챌린지 사업 참여 지자체와 신도시형 도시. 디지털 민원 혁신 예산이 이미 편성되어 있고, AI 도입 의지가 높은 구간입니다.

SOM · 초기 목표

세종 → 인접 중형 도시

본 사업(세종도시관리공단)을 레퍼런스로 확보한 뒤, 유사 규모 도시관리공단으로 확산합니다. 공단 간 벤치마킹 네트워크가 영업 채널이 됩니다.

기존 대안과의 비교 — 「모아」가 이기는 지점

기준안전신문고 (전국 앱)전화 · 방문 민원「모아」
시작 문턱앱 설치 + 본인 인증 필요업무시간 내 통화·방문QR 접속 즉시, 설치·로그인 없음
신고 과정유형을 직접 선택, 긴 양식구두 설명, 위치 특정 어려움사진+위치 3단계, AI가 유형 분류
중복 신고개별 접수 — 담당자가 수동 확인개별 접수유사 신고 통합 + ‘나도 불편해요’
우선순위접수 순서 중심접수 순서 중심위험도+빈도+공감 기반 자동 산정
접근성표준 앱 UI높음(대면) · 단 처리 추적 없음큰 글자·고대비·가이드 MVP 내장
지역 데이터전국 단위 — 지역 특화 어려움데이터화 안 됨지역 특화 라벨 데이터 축적·파인튜닝
※ 안전신문고는 전국 표준 창구로서의 역할이 있으며, 「모아」는 이를 대체하는 것이 아니라 지자체·공단 단위의 현장 처리 효율화 계층을 담당합니다. 확장 로드맵의 다채널 통합 단계에서 기존 창구 접수 건을 관리 화면으로 모으는 연계를 제안합니다.

왜 지금인가

TIMING 1

AI 분류 기술의 상용화

관리형 이미지 인식 API로 초기 구축 비용 없이 수초 내 분류가 가능해졌습니다. 5년 전에는 모델 개발부터 필요했던 일입니다.

TIMING 2

고령화와 디지털 격차

고령 인구 비중이 커질수록 ‘앱 설치형’ 민원 창구의 사각지대가 넓어집니다. 접근성 중심 설계의 가치가 매년 커집니다.

TIMING 3

데이터 행정 요구

감(感)이 아닌 데이터 기반 유지보수 계획 수립 요구가 커지고 있습니다. 통합 신고 데이터는 예방 정비의 근거가 됩니다.

사업화 수익모델 — B2G SaaS 3단 구조

REVENUE 1 · 기반

구축비 + 커스터마이징

지자체·공단별 초기 구축과 조직도·구역 데이터 연동, 기존 시스템 연계 커스터마이징. 첫 계약의 진입 수익이자 구독 전환의 관문입니다.

REVENUE 2 · 성장

월 구독 (SaaS 운영료)

인구·신고량 기준 티어형 구독 — 운영, 유지보수, 모델 업데이트 포함. 예: 소형 공단 월 150만원 ~ 광역 단위 월 500만원 수준. ※ 추정 가격 가설

REVENUE 3 · 심화

데이터 · 모델 부가 서비스

축적 데이터 기반 인프라 노후 분석 리포트(연 단위), 파인튜닝 분류 모델 API 제공, 예방 정비 우선순위 컨설팅.

구조적 마진 개선 — 고객이 늘수록 라벨 데이터가 쌓이고 → 분류 정확도가 오르고 → 담당자 검수 부담과 운영 원가가 내려갑니다. 데이터 플라이휠(7p)이 그대로 수익성 개선 엔진이 됩니다. 신규 지자체 도입 시에도 기축적 모델로 초기 정확도를 높게 시작할 수 있어, 도입 장벽이 계약마다 낮아집니다.
